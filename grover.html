1. 問題: 巨大なデータベースから1つを探す

ソートされていないリストの中から、特定の条件を満たす要素を見つけたいとします。 例えば、100万件のレコードから1件だけマッチするものを探す場合、 古典的なコンピュータにはどれくらいの手間がかかるでしょうか?

たとえ話

電話帳に100万人分の名前がランダムな順番で載っていると想像してください。 「田中太郎」さんを探すには、最悪100万回ページをめくる必要があります。 量子コンピュータなら、たった約1,000回で見つけられます。

2. 古典的な探索の限界

ソートされていない N 個の要素から1つを探す場合、 古典コンピュータは最悪 N回、平均でも N/2回 のチェックが必要です。 これは証明された下界であり、どんなアルゴリズムでも超えられません。

一方、グローバーのアルゴリズムは O(√N) 回で探索を完了します。 N = 1,000,000 なら √N = 1,000。1000倍の高速化です。

10,0005,000回~79回約63倍
1,000,000500,000回~785回約637倍
10125×1011回~106回約50万倍

3. 量子ビットと重ね合わせ

グローバーのアルゴリズムを理解するには、2つの量子力学的な性質が鍵になります。

重ね合わせ (Superposition)

古典的なビットは 0 か 1 のどちらかですが、量子ビット(qubit)は 0 と 1 を同時に持てます。n 個の量子ビットがあれば、 2n 個の状態を同時に表現できます。これにより、 すべての候補を「一度に」考慮することが可能になります。

振幅 (Amplitude)

各状態には「振幅」という複素数が割り当てられています。 測定したときにその状態が観測される確率は、振幅の絶対値の2乗です。 グローバーのアルゴリズムの本質は、正解の振幅を大きくし、不正解の振幅を小さくする操作を繰り返すことです。

数式

n 量子ビットの均等な重ね合わせ状態:
|ψ⟩ = (1/√N) Σ|x⟩    (N = 2n)

各状態の振幅 = 1/√N、観測確率 = 1/N (均等)

5. アルゴリズムの3ステップ

Step 1: 均等な重ね合わせを作る

すべての量子ビットにアダマールゲート (H) を適用し、 N 個すべての状態が等しい振幅 1/√N を持つ状態を作ります。 この時点では、どの状態も同じ確率で観測されます。

Step 2: オラクル (Oracle) — 正解にマークをつける

オラクルは「正解の状態の振幅の符号を反転させる」ブラックボックスです。 正解 |w⟩ の振幅を +a から −a に変えます。 他の状態はそのまま。見た目にはほとんど何も変わっていないように見えますが、 この符号の違いが次のステップで決定的に効いてきます。

たとえ話

暗い部屋に同じ高さの棒が N 本立っていると想像してください。 オラクルは正解の棒だけを上下逆さまにします。 まだ見つけてはいませんが、「印」がつきました。

Step 3: 拡散変換 (Diffusion) — 振幅増幅

ここが魔法の部分です。拡散変換は「平均値を軸にして全ての振幅を反転する」操作です。

正解の振幅は負(平均より下)なので、反転すると大きく跳ね上がります。 不正解の振幅は平均に近いので、反転しても少しだけ減ります。 Step 2 と Step 3 を約 (π/4)√N 回繰り返すと、 正解の振幅がほぼ 1 に、不正解がほぼ 0 になります。

重要ポイント

繰り返しすぎると正解の振幅が下がってしまいます! ちょうど良い回数で止めることが重要で、その最適回数は ⌊(π/4)√N⌋ 回です。

8. 計算量の比較

グローバーのアルゴリズムは二次の高速化 (quadratic speedup) を提供します。 指数的ではないものの、これが最善であることも証明されています。

アルゴリズム クエリ数 種別
線形探索O(N)古典
グローバーO(√N)量子
ソート+二分探索O(N log N) 前処理古典
注意

グローバーの高速化は「ソートされていないデータ」に対するもの。 事前にソートできるなら二分探索 O(log N) の方が速い場合もあります。 グローバーが真価を発揮するのは、構造を持たない探索問題 (暗号解読、最適化問題の候補探索など)です。

9. 応用と限界

応用

暗号解析: AES-128 は古典的には 2128 の試行が必要ですが、 グローバーなら 264 で済みます。これが「ポスト量子暗号」が議論される理由の一つです。

最適化問題: SAT問題や巡回セールスマン問題などで、 候補空間を √N に圧縮する補助アルゴリズムとして使えます。

データベース検索: 理論的にはソートされていないDBの検索にも 使えますが、実際にはデータのロードがボトルネックになる点に注意が必要です。

限界

・二次高速化であり、指数高速化ではない
・オラクルを量子回路として実装するコストがかかる
・現在のNISQデバイスではノイズにより大規模実行が困難
・最適反復回数を超えると成功確率が下がる

10. Qiskitで実装してみる

3量子ビットで |101⟩ を探索する最小限の実装例です。

from qiskit import QuantumCircuit from qiskit_aer import AerSimulator from math import pi, sqrt, floor n = 3 N = 2**n target = 0b101 # 探す状態 qc = QuantumCircuit(n, n) # Step 1: 均等な重ね合わせ qc.h(range(n)) # Grover反復 iterations = floor(pi/4 * sqrt(N)) for _ in range(iterations): # Oracle: |101⟩ の位相を反転 qc.x(1) # ビット1を反転(0→1にマッチ) qc.ccz(0, 1, 2) # 全ビット|1⟩の時に位相反転 qc.x(1) # 元に戻す # Diffusion: 平均周りの反転 qc.h(range(n)) qc.x(range(n)) qc.ccz(0, 1, 2) qc.x(range(n)) qc.h(range(n)) # 測定 qc.measure(range(n), range(n)) # 実行 sim = AerSimulator() result = sim.run(qc, shots=1024).result() counts = result.get_counts() print(counts) # {'101': ~945, ...}
